# V-05 约束落实核查 Agent

> 这是独立的验证步骤，必须由一个全新的 Agent 执行。编排器（写方案的同一主体）不得兼任此步骤——确认偏误风险过高。

## 目标

由一个独立 Agent 通读完整方案 + 约束分发清单，逐条判断每条 H-约束是否在方案对应章节中实际落实。该 Agent 不参与任何章节的撰写，不受"自己的孩子自己夸"的偏误影响。

## 方法

1. **启动一个全新的子Agent**，命名为 `V-05-约束核查`。输入：
   - `output/<name>/临床试验方案_<产品全称>.md`（完整拼装方案）
   - `projects/<name>/约束分发清单.md`（约束→章节映射）
   - `projects/<name>/指导原则约束汇总.md`（约束原文措辞）
   - `projects/<name>/流程检查.md`（A 类标记总表）
   - 输出写入：`output/<name>/V-05核查报告.md`

2. **Agent 动作：**

   ```
   for each H-约束 in 约束分发清单:
     ① 根据"归属章节"定位到方案对应段落
     ② 判断落实程度：
        ✅ 已落实 — 有实质内容呼应约束要求
        ❌ 未落实 — 完全没有对应段落或内容
        ⚠️ 部分落实 — 仅提及条款编号但缺实质操作内容
     ③ 对标注了多章节的跨章节约束，逐章分别检查
   ```

3. **B 类标记核查——A 类标记真实性验证：**

   读 `流程检查.md`，对比 A 类标记与实际产物文件的真实状态：

   ```
   for each 步骤 in 流程检查.md:
     ① 读对应产物文件的实际存在性、行数
     ② 对比 A 类标记中的状态与实际产物
   ```

## 输出格式

```markdown
# V-05 法规约束落实核查报告

> 执行Agent：V-05-约束核查（独立于编排器）
> 核查时间：YYYY-MM-DD

## A 类：逐条约束检查结果
| 约束编号 | 约束要点 | 归属章节 | 检查结果 | 方案行号 | 说明 |

## B 类：流程完整性核查
| 步骤 | A类标记状态 | 实际产物检查 | B类判定 |

## 跨章节约束专项检查
| 约束编号 | 章节A | 章节A状态 | 章节B | 章节B状态 | 说明 |

## 汇总
### A 类：法规约束落实情况
- 全部 H-硬性约束：N 条
- ✅ 已落实：X 条 | ❌ 未落实：Y 条 | ⚠️ 部分落实：Z 条

### B 类：流程完整性 - 全部标记与产物一致：是/否
```

## 判定规则

- 任何 A 类 H-约束标记为 ❌ → HALT
- 任何跨章节约束存在不一致 → ⚠️，标注不一致
- 任何 B 类"流程跳步" → HALT
- 任何 B 类"A 类标记造假" → ❌
- 全部 ✅ 或仅 ⚠️ → 通过

## 禁止的行为

- 禁止编排器自己执行此步骤
- 禁止用关键词 grep 代替逐条段落阅读
- 禁止跳过分布式约束中的任何一条 H-约束
- 禁止跨章节约束只检查一个章节
