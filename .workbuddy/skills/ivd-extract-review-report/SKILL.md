# 审评报告提取器

## 职责

读取 `审评报告/` 目录下的所有 .md 文件（CMDE 官网公开发布的同品种医疗器械技术审评报告），提取已获批同类产品的实际临床试验参数。

审评报告与指导原则、专家共识有本质区别：
- **指导原则**告诉你"药监局要求什么"（应然）
- **专家共识**告诉你"临床上怎么做诊断"（临床实践）
- **审评报告**告诉你"审评员实际批了什么"（实然）

---

## 提取方法

### 第一步：枚举文件

列出 `审评报告/` 目录下的所有文件。

### 第二步：逐文件精读，提取以下五类信息

#### A. 产品基本信息

| 提取项 | 审评报告中典型位置 | 用途 |
|--------|-----------------|------|
| 产品名称 | 报告标题/首页 | 第三章竞品表 |
| 注册人 | "申请人"或"注册人"章节 | 第三章竞品表 |
| 管理类别 | "产品类别"或基本信息 | 对照确认管理类别 |
| 方法学 | "产品结构及组成"或"检验原理" | 同行方法学参考 |
| 样本类型 | "预期用途"或"样本要求" | 对照确认样本类型 |

#### B. 临床试验设计参数（最核心）

| 提取项 | 审评报告中典型位置 | 用途 |
|--------|-----------------|------|
| 临床试验机构数量 | "临床试验"章节 | 第六章机构数参考 |
| 入组总例数 | "临床试验"章节 | 第六章最重要的先例数据 |
| 阳性例数/阴性例数 | "临床试验"章节 | 第六章 |
| 对比试剂名称 | "临床试验"章节 | 第五章对比试剂选择参考 |
| 试验设计类型 | "临床试验"章节 | 第五章设计类型 |
| 评价指标及可接受标准 | "临床试验"章节 | 第五章评价指标 |
| 一致性评价结果 | 阳性符合率/阴性符合率/Kappa | 预期性能参考 |

#### C. 审评关注点（最容易被忽略）

审评报告中"审评关注点""主要问题""发补内容"等小节，逐条提取：
- 审评员提出的问题
- 企业答复的策略
- 发补要求补充的试验

> 同一个审评员可能审你的产品，他之前关注过什么问题，这次大概率也会关注。

#### D. 样本量论证逻辑

- 企业用了什么公式/方法？
- 参数取值（P0、PT 等）的来源和依据？
- 审评员是否接受这个论证？

#### E. 审评结论

- 是否一次性通过？
- 如发补，发补了几轮？
- 发补涉及临床试验的比例？

---

## 输出格式

写入 `projects/<name>/审评报告提取.md`：

```markdown
# 审评报告提取

> 提取时间：YYYY-MM-DD
> 提取文件：X 份

## 一、已获批同类产品概览
## 二、临床试验设计先例汇总
## 三、入组样本量先例（核心参考数据）
## 四、审评关注点与风险预警
## 五、审评先例对当前方案的直接启示
```

---

## 与指导原则、专家共识的协同

- **与指导原则对比**：审评中实际接受的参数是否超出了指导原则的最低要求？
- **与专家共识对比**：审评报告中的入选标准/适应证定义是否与专家共识一致？
