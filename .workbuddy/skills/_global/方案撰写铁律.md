# 方案撰写铁律

> 本文件是所有章节生成的全局行为底线。任何子Agent在开始撰写前，**必须 Read 本文件全文**并逐条遵守。

---

## 规则内容

### 规则 1：尽可能贴合模板，不添加不润色

- **模板没写的不要自己写。** 你加的每一句都可能成为执行端的枷锁或审评端的靶点。
- **模板已有的不要润色。** 合规文件不是文学创作，"优化表达"只会增加出错概率。

### 规则 2：方案管"要多少"，执行管"怎么凑"

- 方案只写统计学论证和法规要求得出的样本量数字。
- 不要替执行端做硬性承诺（如各机构分配量多少例、入组月数多少个月、样本构成比例如何）。
- 方案不是执行方案，不需要告诉执行端怎么做。

### 规则 3：禁止引用低证据材料进入方案正文

以下材料**不得写入方案正文**，仅作为内部验证依据：

| 违禁材料 | 示例 |
|---------|------|
| 竞品入组例数 | "同类产品X入组了Y例" |
| LIS 阳性率/检测量 | "基于N家机构M个月数据" |
| 文献流行病学绝对数字 | "文献报道阳性率为XX%" |
| 审评报告入组数 | "审评报告显示入组了XX例" |

### 规则 4：样本量论证只写四要素

样本量节只出现以下内容：
1. **P₀**（临床可接受标准/目标值）
2. **P_T**（预期值/靶值）
3. **公式**（符号形式，不代入真实值）
4. **计算结果 + 法规引用**

**绝口不提竞品情报。** 竞品入组例数仅用于内部反推 P_T 的参考，不写入方案。

### 规则 5：第三章流行病学背景仅引用定性描述

- 可引用：指导原则中的流行病学描述、专家共识中的定性结论
- **不写入**：文献中的具体阳性率数字、检测量、绝对例数

### 规则 6：占位符清零

- 生成完成后，全文不得残留 `【XXX_*】` 占位符。

---

## 验证方式

每章生成完成后，子Agent自行检查：
1. 方案正文中是否出现竞品例数、LIS数据、文献具体数字 → 如有，删除
2. 是否出现了主Agent给的约束列表中没有的内容 → 如有，确认是否是模板已有的
3. 样本量节是否只包含 P₀+P_T+公式+结果+法规引用 → 如有竞品信息，删除
4. `【XXX_*】` 是否已全部替换 → 如有，补填

> **违反以上任何一条，该章节视为无效，需重新生成。**
