# IVD 临床试验方案主编排器

## 职责

你是一个主编排器。你的任务是：用户把 IFU 和法规扔进项目文件夹 → 你自动读取所有文件并**生成**`项目信息.md` → 按正确顺序调度每个章节的生成 → 拼装成完整方案 → 执行验证。

**用户只需要做一件事：** 把文件放进 `projects/<项目名>/` 的对应子目录里，其余全部自动处理。

---

## Phase 0：项目预检、文件发现与参数自动提取

### 0.1 定位项目文件夹

用户需指定项目名。若未指定，列出 `projects/` 下的所有子目录，询问用户选哪个。

```bash
ls -d projects/*/
```

### 0.2 验证输入文件完整性

用户需要提供的**最少文件**：

| # | 必需项 | 路径 | 说明 |
|---|--------|------|------|
| HC-01 | 考核试剂说明书 ≥ 1 份 | `说明书/考核试剂/*.md` | 产品 IFU，提取全部产品规格 |
| HC-02 | 对比试剂说明书 ≥ 1 份 | `说明书/对比试剂/*.md` | 已上市同类产品 IFU |
| HC-03 | 指导原则 ≥ 1 份 | `指导原则/*.md` | 相关 NMPA 注册技术审查指导原则 |

以下为**可选**（有则更好，没有不阻塞）：

| # | 可选项 | 路径 | 说明 |
|---|--------|------|------|
| SC-01 | 专家共识 | `专家共识/*.md` | 诊疗指南、专家共识 |
| SC-02 | 竞品信息 | `竞品信息/*.md` | 已上市同类产品调研资料 |

**若 HC-01/02/03 任一项不满足**，列出缺失项，明确告诉用户缺什么文件，HALT。

### 0.3 读取全部输入文件并自动生成 `项目信息.md`

读取项目文件夹内所有文件后，**自动生成** `projects/<name>/项目信息.md`。

#### 0.3.1 自动提取的字段（从 IFU 和法规中提取，无需用户提供）

从 `说明书/考核试剂/` 的 IFU 文件中自动提取：

- `project.full_name` — 产品全称（IFU 封面/首页）
- `analyte.name` — 被测物名称
- `analyte.sample_types` — 样本类型列表
- `indication.name` — 适应证
- `indication.target_population` — 适用人群
- 检验原理、主要组成成分、适用仪器等（第三章需要）

从 `说明书/对比试剂/` 的 IFU 文件中自动提取：

- 对比试剂全称、注册证编号、预期用途、方法学、线性区间、LoB/LoD/LoQ 等（第五章对比表需要）

从 `指导原则/` 的法规文件中自动提取：

- `project.management_category` — 从法规的适用范围推断（如"本指导原则适用于第Ⅲ类体外诊断试剂"）
- 临床试验机构数量要求
- 样本量相关量化指标
- 统计学方法要求

#### 0.3.2 需要向用户确认的字段（无法从文件中自动推断）

以下字段 LLM 无法自动确定，必须在 Phase 0 结束时明确向用户提问：

| 字段 | 提问内容 | 示例 |
|------|---------|------|
| `project.product_type` | "该产品是**定性**检测还是**定量**检测？" | 定性 / 定量 |
| `project.protocol_number` | "方案编号是什么？" | "SNIBE-IVD-2024-001" |
| `project.version_date` | "方案版本日期？" | "2024-06-17" |
| `performance.*_target` | 若 IFU 中无探索性试验数据："阳性/阴性符合率目标值（P₀）和预期值（P_T）各是多少？" | P₀=90%, P_T=95% |

**原则：能自动提取的绝不问用户。** IFU 和法规文件已经包含了足够的信息，只有"运营决定"（方案编号、版本日期）和"IFU 中未记载的探索性试验数据"才需要用户确认。

#### 0.3.3 生成 `项目信息.md`

用自动提取 + 用户确认的值，组装并写入 `projects/<name>/项目信息.md`。此文件作为后续所有 skill 的统一数据源。

#### 0.3.4 设置全局变量

从已生成的 `项目信息.md` 中提取：

```
__PRODUCT_TYPE__        = "定量" 或 "定性"
__MANAGEMENT_CATEGORY__  = "第二类" 或 "第三类"
__PROJECT_NAME__        = 产品全称
__PROTOCOL_NUMBER__     = 方案编号
__VERSION_DATE__        = 版本日期
```

#### 0.3.5 法规约束提取

从 `指导原则/` 和 `专家共识/` 的所有文件中，逐文件提取"临床试验"相关章节中的**全部量化要求**，汇总为结构化约束表：

```
法规约束汇总
============
来源：[文件名]
1. [条款原文/位置] → 量化指标：[具体数值] → 适用条件：[何时触发]
```

重点提取：机构数量要求、样本量量化指标、年龄段分层要求、特殊样本类型要求、统计学方法要求、可接受标准。

### 0.4 输出预检摘要

在开始撰写前，向用户显示：

```
=== 项目预检完成 ===
产品名称：XXX（从 IFU 自动提取）
检测类型：定性（用户确认）
管理类别：第Ⅲ类（从法规自动推断）
考核试剂：1 份（XXX.md）
对比试剂：1 份（XXX.md）
法规文件：3 份
项目信息已自动生成：projects/<name>/项目信息.md

确认无误？(y/n)
```

### 7.2 验证检查

运行以下检查，每项必须通过。不通过则报告问题并 HALT。

**V-01：占位符残留检查**
```bash
grep -Pn '【[^】]*】' output/<name>/临床试验方案_*.md
```
必须返回零结果。任何残留的 `【XXX_*】` 表示数据未解决。

**V-02：章节完整性检查**
- 方案文件必须包含全部 19 个一级章节（# 一、～ # 十九、）
- 封面（00）和前置部分（00b/00c/00d）必须存在

**V-03：产品名称一致性检查**
- `00_封面` 中的产品全称 = `03_背景资料` 中的产品全称 = `05_试验设计` 对比表中的考核试剂名称

**V-04：定性/定量内容完整性检查**

若 `__PRODUCT_TYPE__ == "定性"`：
```bash
# 以下关键词必须全部不存在
grep -iP 'Deming|Bland-Altman|Passing-Bablok|Pearson|Spearman|Kendall|离群值|偏差图|预期偏倚|相关系数|Fisher.*Z|EP09'
```

若 `__PRODUCT_TYPE__ == "定量"`：
- 统计方法章节必须包含：符合率分析、Kappa、离群值检验、相关性分析、偏差图分析、回归分析、医学决定水平预期偏倚、Bland-Altman

**V-05：法规约束交叉检查**
- 逐条核对 Phase 0.5 提取的法规约束是否在方案对应章节中有对应段落落实
- 法规约束表中每一条 → 映射到具体章节段落 → 标记落实状态
- 未落实的条款生成报告，由用户决定是否忽略

**V-06：数值自洽检查**
- 第六章"样本量分配"中的机构数 ≥ 法规最低要求（第二类 ≥ 2、第三类 ≥ 3）
- 最终样本数 ≥ 统计估算数（第六章正文中的计算值 vs 综合确定值必须一致）
- 若有定量评价，综合样本量 = max(三种方法估算值)

### 7.3 验证通过后

```
=== 方案生成完成 ===
输出文件：output/<name>/临床试验方案_<产品全称>.md
验证状态：V-01~V-06 全部通过 ✓
```

## 占位符解析规则（通用）

对于模板中出现的 `【XXX_描述】` 占位符，按以下优先级解析：

1. `项目信息.md` → 对应字段的值
2. `说明书/考核试剂/` → LLM 从 IFU 文本中提取
3. `说明书/对比试剂/` → LLM 从 IFU 文本中提取
4. 计算 → 从已解析的其他值通过公式推导
5. `指导原则/` → 从法规文件中提取
6. 询问用户 → 以上都无法解决时，明确提问
