# 审评报告提取器

## 职责

读取 `审评报告/` 目录下的所有 .md 文件（CMDE 官网公开发布的同品种医疗器械技术审评报告），提取已获批同类产品的**实际临床试验参数**。

审评报告与指导原则、专家共识有本质区别：
- **指导原则**告诉你"药监局要求什么"（应然）
- **专家共识**告诉你"临床上怎么做诊断"（临床实践）
- **审评报告**告诉你"审评员实际批了什么"（实然）——这是最有参考价值的先例信息

---

## 审评报告的来源和价值

CMDE（国家药品监督管理局医疗器械技术审评中心）在官网公开发布已获批产品的技术审评报告。下载后转为 .md 放入 `审评报告/` 目录。

价值：
1. **入组样本量先例** — 同类产品审评通过时的实际入组数，是样本量论证的最强参考
2. **临床试验设计标杆** — 审评员接受的设计类型、对比试剂选择、评价指标设置
3. **审评关注点** — 审评报告中"审评关注点"或"主要问题"部分，揭示了同类产品审评中最被关注的环节
4. **补充试验要求** — 有些报告记录了发补时要求补充的试验，这些都是需要提前规避的坑

## 提取方法

### 第一步：枚举文件

```bash
ls projects/<name>/审评报告/
```

### 第二步：逐文件精读，提取以下五类信息

#### A. 产品基本信息

| 提取项 | 审评报告中典型位置 | 用途 |
|--------|-----------------|------|
| 产品名称 | 报告标题/首页 | 第三章竞品表 |
| 注册人 | "申请人"或"注册人"章节 | 第三章竞品表 |
| 管理类别 | "产品类别"或基本信息 | 对照确认管理类别 |
| 方法学 | "产品结构及组成"或"检验原理" | 同行方法学参考 |
| 样本类型 | "预期用途"或"样本要求" | 对照确认样本类型 |

#### B. 临床试验设计参数（最核心）

| 提取项 | 审评报告中典型位置 | 用途 |
|--------|-----------------|------|
| 临床试验机构数量 | "临床试验"章节 | 第六章（一）2 参考 |
| 入组总例数 | "临床试验"章节 → "共入组XX例" | 第六章（一）1 最重要的先例数据 |
| 阳性例数/阴性例数 | "临床试验"章节 | 第六章（一）1 |
| 对比试剂名称 | "临床试验"章节 → "以XX为对比试剂" | 第五章对比试剂选择参考 |
| 试验设计类型 | "临床试验"章节 → "采用……设计" | 第五章（一）1 |
| 评价指标及可接受标准 | "临床试验"章节 | 第五章（三） |
| 是否纳入干扰/交叉反应样本 | "临床试验"章节 | 第五章（二） |
| 一致性评价结果 | "临床试验"章节 → 阳性符合率/阴性符合率/Kappa | 预期性能参考 |

#### C. 审评关注点（最容易被忽略）

审评报告中常有"审评关注点""主要问题""发补内容"等小节。逐条提取：

| 提取项 | 说明 |
|--------|------|
| 审评员提出的问题 | 这些问题很可能也在我们的方案中被问到 |
| 企业答复的策略 | 参考其论证思路 |
| 发补要求补充的试验 | 提前规避——在初始方案中就包含这些内容 |

> ⚠️ 这些信息极为宝贵。同一个审评员可能审我们的产品，他之前关注过什么问题，这次大概率也会关注。

#### D. 样本量论证逻辑

审评报告有时会描述企业的样本量论证过程和审评员的接受/质疑：

- 企业用了什么公式/方法？
- 参数取值（P₀、P_T 等）的来源和依据是什么？
- 审评员是否接受这个论证？

> **用途**：第六章样本量论证时，可以直接引用同品种审评先例作为"行业惯例"参考。

#### E. 审评结论

- 是否一次性通过？
- 如发补，发补了几轮？
- 发补涉及临床试验的比例是多少？

## 与指导原则、专家共识的协同

审评报告提取完成后，与另外两份提取报告交叉参考：

- **与指导原则对比**：审评中实际接受的参数是否超出了指导原则的最低要求？如有，标注"审评实践中要求高于指导原则最低线"
- **与专家共识对比**：审评报告中的入选标准/适应证定义是否与专家共识一致？
