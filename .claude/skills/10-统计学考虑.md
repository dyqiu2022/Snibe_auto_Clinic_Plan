## ⚠️ 样本量确定的战略逻辑（厂方立场）

> **你是申办者（厂家），不是中立的统计师。** 你的目标是在确保审评通过的前提下，把样本量压到最低。

### 核心张力

```
统计公式: P₀=85%, P_T=90% → n=200
统计公式: P₀=90%, P_T=95% → n=350
法规要求: "应采用统计学方法估算样本量"（模糊）
审评员心理: "这个产品类型，没个300例过不了"（不成文规则）
竞品IFU情报: 竞品A入组240例（2023年获批）← 这个数字审评批过了
厂方目标: 贴近240例，但方案里不能写"因为竞品做了240所以我们也做240"
```

### 实战策略：竞品情报 → 反向推参数 → 干净的统计学论证

**这个策略是 IVD 临床试验方案写作的核心技巧：**

#### 第1步：从竞品情报确定"靶心"

读取 `产品信息提取.md` → "八、样本量情报汇总"，找到：
- 最近获批（≤2年）的同类产品中，**最低**的入组数
- 这个数字不需要是最低的——关键是"审评确实批过这个数量"
- 记作 `N_competitor`

从 `审评报告提取.md` 中（若存在）：
- 审评报告样本量通常偏高 → 不作为靶心
- 但审评关注点中关于样本量的讨论 → 极有价值（知道审评员在意什么）

#### 第2步：选定"看起来常规"的 P₀

P₀ 是目标值（临床可接受标准下限）。选一个行业里常用的值，让它"看起来正常"：

| 产品类型 | 常用 P₀ | 说明 |
|---------|---------|------|
| 定性试剂 | 85%~90% | 符合率目标值 |
| 定量试剂 | 相关系数 ρ₀=0.975 | EP09-A3 要求 |

**P₀ 不能太极端**——如果 P₀=80%，审评员会质疑"你对产品这么没信心？"；如果 P₀=95%，n 会很大，违背控成本的目的。

#### 第3步：反推 P_T

用选定的 P₀，反向计算：**什么样的 P_T 能让 n 刚好 > N_competitor 且看起来合理？**

```
给定: α=0.05, β=0.20, P₀=90%
目标: n ≥ N_competitor（且略高出一点，如 +5%~+10%）
反推: P_T = ?
```

P_T 是"预期值"——你预期产品能达到的性能。如果反推出来的 P_T 在合理范围内（比如 94%~97%），这个参数组合就是可用的。如果反推出来 P_T=99% 太高（审评员会质疑"你凭什么预期这么高"），则调低 P₀。

#### 第4步：写方案——只留干净的统计学脚印

方案正文的写法：

> "根据临床需求，阳性符合率目标值（P₀）设置为不低于 90.0%。根据考核试剂与已上市同类产品比对的探索性试验结果，考核试剂预期阳性符合率（P_T）可达到 95.0%。将 α=0.05, β=0.20 代入样本量估算公式，算得阳性组最低样本量为 XXX 例……"

**方案里不要出现任何一个字关于竞品样本量。** 方案里只有：
- P₀（选自行业常用值）
- P_T（来自"探索性试验"——反推出来的值）
- 计算结果（刚好在你想要的靶心附近）

**竞品情报的作用是让你知道"什么数字能过"——它不是论据，它是指南针。**

#### 第5步：若无法找到竞品情报

如果 `产品信息提取.md` 中没有竞品样本量情报（竞品 IFU 太旧、找不到、未提供）：

1. 仅依赖统计公式
2. 选择保守的 P₀（取常用范围下限）
3. 在方案中额外加强统计论证的严密性（展示多种方法、使用精确概率法等）
4. **但不要轻易把 n 压得太低**——没有竞品参考意味着你不知道及格线在哪，需要更保守
5. 在输出摘要中 WARN 用户："缺乏竞品先例参考，样本量论证仅基于统计学公式，存在审评风险"

### ◇ 定性/定量闸门：样本量估算

#### 定量产品（`__PRODUCT_TYPE__ == "定量"`）

使用**三种**方法分别估算样本量：

**方法(1)：基于相关系数的样本量估算**

公式（Dixon & Massey）：

$$n=\frac{(Z_{1-\alpha/2}+Z_{1-\beta})^{2}}{[FZ(\rho_{1})-FZ(\rho_{0})]^{2}}+3$$

$$FZ(\rho)=\frac{1}{2}\ln\left(\frac{1+\rho}{1-\rho}\right)$$

$$n'=\frac{n}{(1-f)}$$

参数来源：
- `ρ₁` = `performance.expected_correlation`（来自探索性试验/项目信息）
- `ρ₀` = `performance.min_correlation`（EP09-A3 要求 ≥0.975；若项目信息未指定则默认 0.975）
- `α` = `statistical.alpha`（默认 0.05）→ Z₁₋α/₂ = 1.96
- `1-β` = `statistical.power`（默认 0.80）→ Z₁₋β = 0.8416
- `f` = `performance.dropout_rate`（默认 0.05）

计算步骤：
1. 计算 Fisher's Z 变换：FZ(ρ₁) 和 FZ(ρ₀)
2. 代入公式得 n
3. 调整脱落率：n' = ceil(n / (1-f))
4. 输出：统计估算 n 例 → 预计入组 n' 例

**方法(2)：基于 Bland-Altman 一致性分析的样本量估算**

采用 Lu et al. 方法，5 参数输入：
- `α` = `statistical.alpha`（默认 0.05）
- `β` = `statistical.beta`（默认 0.20）
- `μ` = `performance.bias_mean`（预期偏倚均值，%）
- `σ` = `performance.bias_sd`（预期偏倚标准差，%）
- `δ` = `performance.acceptable_bias_pct`（临床可接受的最大偏倚，%）

优先用 MCP 工具或 MedCalc 计算；若不可用则 LLM 按 Lu et al. 公式计算。
输出：统计估算 n 例 → 预计入组 n' 例。

**方法(3)：基于阳性符合率和阴性符合率的样本量估算**

单组目标值法公式：

$$n=\frac{\left[Z_{1-\alpha/2}\sqrt{P_{0}(1-P_{0})}+Z_{1-\beta}\sqrt{P_{T}(1-P_{T})}\right]^{2}}{(P_{T}-P_{0})^{2}}$$

参数来源：
- `P₀(阳性)` = `performance.pos_agreement_target`
- `P_T(阳性)` = `performance.pos_agreement_expected`
- `P₀(阴性)` = `performance.neg_agreement_target`
- `P_T(阴性)` = `performance.neg_agreement_expected`
- `α` = 0.05 → Z₁₋α/₂ = 1.96
- `β` = 0.20 → Z₁₋β = 1.84

分别计算阳性组 n_pos 和阴性组 n_neg，总 n = n_pos + n_neg。
输出：阳性组 ≥ n_pos 例、阴性组 ≥ n_neg 例、预计总入组 = ceil(n/(1-f)) 例。

**综合确定（定量产品）：**

取三种方法估算结果的最大值：
```
最终阳性数 = max(方法1阳性, 方法2不区分阴阳性, 方法3阳性)
最终阴性数 = max(方法1阴性, 方法2不区分阴阳性, 方法3阴性)
最终总数 = ceil((最终阳性数 + 最终阴性数) / (1 - f))
```

同时对照法规约束（Phase 0.5 提取的量化要求），若法规要求高于统计估算值，取法规值。

最终在"综合考虑"段落明确写出"统计估算值 vs 法规最低值 → 取最大值"的逻辑链。

## 隐性规则

### IR-06-01: 样本量向上取整

所有计算出的样本量一律 ceil() 向上取整，不得出现小数。

### IR-06-02: 法规 > 统计

当法规明确要求的最低样本量 > 统计公式估算值时，以法规要求为准。必须在正文中明确展示"统计估算 X 例 < 法规最低 Y 例 → 最终确定为 Y 例"的逻辑。

### IR-06-03: Z 值精度

- Z₁₋α/₂ (α=0.05, two-sided) = 1.96 精确
- Z₁₋β (β=0.20) = 0.8416（保留 4 位小数）

### IR-06-04: EQA 标准年份

若法规文件夹中有比 `eqa.standard_name` 更新年份的 EQA 标准，使用更新版本。从文件名/内容中提取年份。
