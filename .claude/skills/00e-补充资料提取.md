# 补充资料提取器（文献调研 + 医院LIS数据分析）

## 职责

读取 `补充资料/` 下的两类高价值补充资料，提取对临床试验方案撰写有用的信息：

- **文献调研** → 充实第三章研发背景（被测物生物学意义、疾病机制、最新进展）
- **医院LIS数据分析** → 🔑 **本地真实世界证据**，是样本量可行性论证的核心

> **LIS 数据的特殊价值**：竞品 IFU 告诉你"别人做了多少例"（外部先例），专家共识告诉你"疾病怎么诊断"（临床规范），而 **LIS 数据告诉你"在本地医院，这个被测物有多少阳性样本可入组、浓度分布如何"（内部可行性）**。审评员非常看重这一条——很多方案被发补正是因为"无法证明阳性样本的可及性"。

## 二、医院 LIS 数据分析提取 🔑 核心价值

读取 `补充资料/` 下的文件，**按内容识别哪些是 LIS 数据分析**（关键词：检测量、阳性率、浓度分布、LIS、信息系统、统计时段等），对其提取以下信息。

### ⚠️ LIS 数据的张力与隐性知识

LIS（实验室信息系统）数据是医院历史检测的真实记录，对方案撰写极有价值，但也有必须注意的"门道"：

#### 张力 1：本地代表性 vs 人群代表性

- **价值**：LIS 是本地真实世界数据，审评员认可度高，比引用全国文献更有针对性
- **风险**：单家医院的 LIS 可能不代表目标人群（如三甲医院危重症多、社区医院轻症多）
- **做法**：若 LIS 来自多家机构，标注"基于 N 家机构汇总"；若仅 1 家，在论证中说明代表性，并配合入组多家机构来弥补

#### 张力 2：时效性

- LIS 数据的**采集时间窗口**很关键
- 例如：呼吸道病原体的 LIS 数据，疫情前后阳性率差异巨大；流感数据有明显季节性
- **做法**：提取时必须记录数据时间跨度（如"2023年1月-2024年12月"），并在论证中考虑时效性

#### 张力 3：数据脱敏与合规

- LIS 含患者隐私，**方案中只能引用汇总统计**（率、分布、计数），严禁出现可识别个体的数据
- **做法**：提取时只保留汇总层面的数字，丢弃任何个体记录

### 必须提取的 5 类信息

| # | 类别 | 提取内容 | 方案用途 |
|---|------|---------|---------|
| S-01 | **检测总量** | 时间跨度内总检测例数、月均检测量 | 第六章样本量可行性论证 |
| S-02 | **阳性率/异常率** | 阳性比例、按病原体/指标的分层阳性率 | P_T 取值依据 + 阳性样本可及性 |
| S-03 | **浓度/Ct值分布** | 定量产品的浓度分布（覆盖范围、中位数、分位数）；定性PCR的Ct值分布 | "覆盖线性范围""弱阳性样本"法规要求的可行性 |
| S-04 | **人群分布** | 年龄/性别/科室分层分布 | 第五章入选标准可行性、第六章分层分析可行性 |
| S-05 | **样本类型分布** | 各样本类型（血清/血浆/咽拭子等）的占比 | 第五章样本类型、多种样本类型分别分析的可行性 |

### LIS 数据如何支撑第六章样本量论证 🔑

> ⛔ **LIS 数据仅供内部决策，不得写入方案正文。** 方案正文只呈现统计学公式计算结果（P₀ + P_T + n），不出现 LIS 数据、竞品信息、文献流行病学数字等低证据等级材料。详见 CLAUDE.md 铁律。

这是 LIS 数据最核心的用途——**在幕后帮你判断可行性，不是在台前帮你写方案**。逻辑链：

```
LIS 数据：N 家机构、M 个月内，该被测物共检测 X 例，阳性率 Y%，其中弱阳性约占 Z%
    ↓
【内部判断】推算：联合这些机构、计划入组周期 T 个月
    → 预计可获得阳性样本 = X × Y% × (T/M) × 调整系数
    ↓
【内部判断】与第六章统计估算的阳性样本需求对比
    → 若预计可获得 ≥ 统计需求 → 样本量参数可行，放心用
    → 若预计可获得 < 统计需求 → 调整 P_T 重新算，或增加机构数/延长周期
    ↓
【方案正文只写】干净的统计学论证（P₀ + P_T + 公式 + n），绝口不提 LIS
```

同样，LIS 的浓度分布数据用来**内部验证**法规隐藏要求的可行性：
- "应覆盖线性范围" → LIS 显示样本浓度跨越 a~b 范围 ✓ → 内部确认可行，不在方案中引用 LIS 数值
- "应包含弱阳性样本不少于 X 例" → LIS 显示接近 cutoff 的样本占 Z% → 内部确认可筛选，方案中只写要求不写"我们有"

> ⛔ **以下标注"内部用"的段落仅存于 `补充资料提取.md`，不得写入方案正文。**

在 `补充资料提取.md` 的 LIS 部分：

```markdown

- 时间跨度：YYYY-MM ~ YYYY-MM（共 M 个月）

### 关键统计 ⛔ 内部用

| 指标 | 数值 | 分层 |
|------|------|------|
| 总阳性率 | Z% | 按病原体/指标分：A=..%, B=..% |
| 浓度分布 | 中位数 .., 范围 ..~.. | 弱阳性占 ..%, 中阳性 ..%, 强阳性 ..% |
| 年龄分布 | <5岁 ..%, 6-21 ..% ... | 满足分层分析要求 |

### 对法规隐藏要求的可行性内部判断 ⛔ 内部用
- 覆盖线性范围：LIS 浓度范围 ..~.. ✓
- 弱阳性样本可得性：接近 cutoff 样本占 ..% ✓
- 各年龄段分层：LIS 各年龄段均有分布 ✓
```

- `补充资料/` 为空，或仅含文献无 LIS 数据 → 第三章背景依赖专家共识 + 文献，第六章样本量可行性论证缺少本地证据
- **WARN 用户**：审评可能就样本可及性发补，建议补充医院 LIS 数据分析
- 不阻塞流程，但在摘要中提示增强空间

> 读取时**按文件内容判断类型**（文献 vs LIS 数据），而非依赖子目录分类。用户可把各类相关文件平铺放入 `补充资料/`，无需预先分类。
