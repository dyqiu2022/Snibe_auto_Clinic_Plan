# 说明书与竞品信息提取器

## 职责

读取 `说明书/考核试剂/`、`说明书/对比试剂/`、`竞品信息/` 下所有文件，从中提取全部与临床试验方案撰写相关的产品参数，输出结构化提取报告。

说明书是 IVD 临床试验方案最核心的输入——方案中第三章（产品基本信息）、第五章（对比表、入排标准、评价指标）、第六章（样本量参数）的大量字段直接来源于 IFU。**IFU 读漏一个参数，方案就可能缺关键信息。**

---

## 执行方法

### 第一步：枚举文件

```bash
ls projects/<name>/说明书/考核试剂/
ls projects/<name>/说明书/对比试剂/
ls projects/<name>/竞品信息/        # 若存在
```

### 第二步：逐文件精读

对每个 IFU 文件，按以下结构逐项提取。IFU 通常按 NMPA 说明书格式编写，各章节有固定标题。

### 第三步：输出提取报告

写入 `projects/<name>/产品信息提取.md`。

## 隐性知识 — IFU 解读的常见陷阱

### 陷阱 1：方法学术语不统一

不同厂家的 IFU 对同一方法学可能有不同措辞：
- "化学发光免疫分析法" = "化学发光法" = "CLIA" = "直接化学发光法"
- "PCR-荧光探针法" = "荧光PCR法" = "实时荧光PCR法" = "qPCR法"

**做法**：提取时保留 IFU 原文措辞，在括号中标注标准化术语。第五章对比表中使用标准化术语以便比较。

### 陷阱 7：阳性判断值的类型

定性试剂：IFU 给出的是 cutoff 值（如 Ct≤38 为阳性）
定量试剂：IFU 给出的是参考区间（如 <100 pg/mL 为正常）

**判断产品是定性还是定量的最直接依据**。提取后立即对比 `__PRODUCT_TYPE__`。

## 竞品信息提取

若 `竞品信息/` 目录非空：

1. 从每个竞品文件中提取：产品名称、注册人、注册证编号、方法学、样本类型、预期用途
2. 若竞品文件是审评报告或文献，额外提取：**实际入组样本量**（极有价值的参考数据）
3. 构建竞品汇总行，可用于第三章表1

若 `竞品信息/` 为空：
- 标注 "竞品信息待补充"
- 第三章表1 将仅包含对比试剂一行 + 占位符行
- 提醒用户：补充竞品信息可提升方案论证质量

---

## 对比试剂选择论证的隐性知识

第五章需要论证"为什么选这个对比试剂"。从 IFU 中提取以下论据：

1. **方法学可比性**：考核和对比的方法学是否相近（如同为化学发光法）→ 高可比性
2. **预期用途一致性**：适应症和适用人群是否重叠 → 高一致性
