# IVD 临床试验方案撰写项目

本项目用于体外诊断试剂（IVD）临床试验方案的起草与撰写。所有方案严格遵循国家药品监督管理局相关法规要求，基于模板结构生成，并根据定性/定量检测产品的特点进行适配。

## 项目概述

- 申办者：深圳市新产业生物医学工程股份有限公司
- 产品类型：体外诊断试剂（PCR荧光探针法等多重核酸检测试剂、化学发光免疫检测试剂等）
- 输出格式：Markdown (.md)
- 方案语言：中文

## 工作方式：Skills 集群 + 项目文件夹

### 架构

本系统采用三层架构：

1. **CLAUDE.md（本文件）** — 全局配置和工作流入口
2. **Skills 集群（`.claude/skills/`）** — 24 个 chapter skill + 1 个主编排器，每个 skill 是**完全项目无关的抽象指令**
3. **模板层（`templates/方案模板/`）** — 23 个章节模板文件，含 `【XXX_描述】` 占位符，仅被 skills 引用读取

Skill 不硬编码任何产品名称、病原体、法规条款的具体数值。所有项目特异性数据放在 `projects/<项目名>/` 下，skill 按约定路径自动发现并处理。

### 项目文件夹结构

```
projects/<项目名>/
├── 项目信息.md            # ⚡ 自动生成（主编排器 Phase 0 从 IFU+法规中提取）
├── 说明书/
│   ├── 考核试剂/                  # ← 用户放入：考核试剂 IFU（1份或多份 .md）
│   ├── 对比试剂/                  # ← 用户放入：本试验直接比对的对比试剂 IFU
│   └── 其他同品种试剂盒说明书/    # （可选）其他已上市同类产品正式 IFU（竞品表+样本量情报最优来源）
├── 指导原则/              # ← 用户放入：产品相关的注册技术审查指导原则 .md
├── 专家共识/              # （可选）诊疗指南、专家共识 .md
├── 竞品信息/              # （可选）非正式IFU的竞品调研资料 .md
├── 审评报告/              # （可选）CMDE 同品种技术审评报告 .md
└── 补充资料/              # （可选）文献调研、医院LIS数据分析等杂项相关文件平铺放入
```

### 如何使用

**新建项目（一键脚手架）：**

```bash
bash scripts/init_project.sh <项目名>
# 自动创建 projects/<项目名>/ 下 6 个标准子目录 + 项目信息.md 模板
```

然后把输入文件放入对应子目录：
- `说明书/考核试剂/` — 考核试剂 IFU（.md 格式）【必填】
- `说明书/对比试剂/` — 本试验直接比对的对比试剂 IFU【必填】
- `说明书/其他同品种试剂盒说明书/` — 其他已上市同类产品正式 IFU（可选，样本量情报最优来源）
- `指导原则/` — 相关 NMPA 注册技术审查指导原则（.md 格式）【必填】
- `专家共识/` — 诊疗指南、专家共识（可选）
- `竞品信息/` — 非正式 IFU 的竞品调研资料（可选）
- `审评报告/` — CMDE 同品种技术审评报告（可选）
- `补充资料/` — 文献调研、医院LIS数据分析等杂项相关文件平铺放入（可选，LIS数据是样本量可行性论证核心证据）

**生成方案：**

1. 输入：`/ivd-00-orchestrator 项目名=<name>`
2. 主编排器自动完成：
   - Phase 0：调用 4 个预处理 skill（00a 指导原则 / 00b 说明书 / 00c 专家共识 / 00d 审评报告）→ 自动生成 `项目信息.md`
   - 对无法推断的字段（方案编号、版本日期、定性/定量类型、P₀/P_T 等）向用户确认
   - 调度全部 23 个 chapter skill 逐章撰写
   - 最终拼装验证，输出到 `output/<项目名>/临床试验方案_<产品全称>.md`
3. 也可单独调用某一章：`/ivd-10-statistics` 只生成第六章

### MCP 工具

项目内置样本量计算 MCP Server（`mcp/sample_size_server.py`），通过项目级 `.mcp.json` 自动加载：

- **首次使用**：运行 `bash scripts/setup.sh`（安装 uv + 预拉依赖），详见 `mcp/README.md`
- 从 GitHub 克隆后无需手动配置，uv run 自动建环境
- **覆盖范围**：仅样本量计算（方案撰写阶段唯一需要确定性计算的环节）
  - `calc_correlation_sample_size` — 相关系数法（定量）
  - `calc_bland_altman_sample_size` — Bland-Altman 法（定量）
  - `calc_agreement_sample_size` — 符合率法（定性/定量）
  - `validate_sample_size_logic` — 样本量逻辑链验证
- 第六章样本量计算必须优先调用这些工具，确保结果确定性

> **说明**：方案阶段不封装 Wilson 置信区间/Kappa/卡方等统计计算 MCP——这些需要临床试验四格表数据，属于**临床试验报告撰写**阶段（另一项目范畴），方案阶段只写方法名、不算数值。

## 定性/定量适配规则

当考核试剂为**定性**检测产品时，skills 自动执行以下适配（由各 skill 的"删除清单"和"新增清单"控制）：

### 必须删除的定量专属内容

- 回归分析（Deming回归、Passing-Bablok回归）
- Bland-Altman一致性分析
- 偏差图分析（数值偏差图、排序偏差图）
- 离群值检验
- 相关性分析（Pearson、Spearman、Kendall）
- 医学决定水平处的预期偏倚分析

### 必须保留或新增的定性专属内容

- 阳性符合率、阴性符合率、总符合率（每种待测指标分别计算）
- Kappa一致性检验（每种待测指标分别计算）
- 混合感染/多重阳性样本分析（多重检测产品必须）
- 各年龄段分层分析（按法规要求：<5岁、6-21岁、22-59岁、>60岁）
- 四格表卡方检验
- 不一致结果确认方法（测序、第三方试剂、临床综合判断）

## 法规处理规范（Skills 设计参考）

### 常见错误

AI 读完法规文件后容易犯以下错误：

- 只关注法规的总体框架和大意，忽略其中的**具体量化要求**
- 读了法规但未能将法规条款精确映射到方案的具体段落
- 遗漏法规中"临床试验"章节里的细节性要求（如样本量计算方法、分层分析、特殊样本类型等）

### 正确做法

对每份法规文件，**仅提取"临床试验"或"临床评价"相关章节**，将其中所有具体要求逐条列出：

- 提取**所有量化指标**（如：阳性≥X例、阴性≥Y例、弱阳性≥Z例）
- 提取**所有必须执行的条款**（如：≥N家临床试验机构、年龄段分层分析）
- 提取**具体的统计学方法要求**（如：使用什么公式、什么置信区间标准）
- 提取**样本相关要求**（如：样本类型、采集方式、保存条件）
- 提取**特殊要求**（如：混合感染分析、交叉反应样本、竞争性干扰）

## 方案格式规范

- 标题层级：`#` 一级标题、`##` 二级标题、`###` 三级标题，每级标题加粗
- 表格使用Markdown格式（`|---|---|`）
- 公式使用LaTeX数学语法（`$$...$$`）
- 缩略语、统计学符号使用斜体
- 参考文献使用序号方括号格式
- 占位符使用`【XXX_描述】`格式标注需要后续填写的内容
