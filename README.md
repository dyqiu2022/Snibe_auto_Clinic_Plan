# IVD 临床试验方案自动撰写系统

> WorkBuddy 桌面版 · 一句话启动，零命令行操作

## 这是什么

一个**自动生成 IVD（体外诊断试剂）临床试验方案**的 AI 工作流。

**适用人群**：申办者（厂家）法规部、临床试验负责人、CRA、医学事务。

**适用产品**：PCR-荧光探针法多重核酸检测试剂、化学发光免疫检测试剂等定性/定量 IVD 试剂。

| 开始一个新产品 | "开始新项目：XXX 试剂盒" |

## 工作流程（5 分钟看完）

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ Phase 0  │ → │ Phase 1  │ → │ Phase 2-4│ → │ Phase 5-6│ → │ Phase 7  │
│ 项目预检 │   │ 简单章节 │   │ 复杂章节 │   │ 缩略语+  │   │ 拼装+    │
│ 文件发现 │   │ 并行生成 │   │ 串行生成 │   │ 参考文献 │   │ 验证导出 │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
```

### Phase 0：项目预检
- 你把 IFU 和指导原则放进 `projects/<项目名>/` 的对应子目录
- AI 自动读取、提取参数、生成 `项目信息.md`
- AI 给你一份"预检摘要"，你确认后进入下一阶段

### Phase 1：简单章节（18 个）
- 封面、申办者信息、监查计划等——大部分是纯模板，AI 直接透传
- **耗时 5 分钟**

### Phase 2-4：复杂章节（3 个）
- 第三章 背景资料
- 第五章 试验设计
- 第六章 统计学考虑
- 这三章有大量数据填充，AI 严格按你的 IFU + 法规约束生成
- **耗时 30 分钟**

### Phase 5-6：后置章节
- 缩略语（AI 扫所有章节自动收集）
- 参考文献（AI 整理所有引用）
- 目录（AI 扫描标题自动生成）

### Phase 7：拼装 + 验证 + 导出
- AI 把 19 个章节拼成完整 .md
- AI 自动跑 V-01~V-05 验证（占位符、完整性、名称一致性、定性/定量禁忌词、约束落实）
- AI 把 .md 转成 .docx（用 python-docx，三线表、中文字体都正常）

## 协作模式（多人参与修改 Skills）

适合法规、统计、临床团队的**非技术同事**。

### 场景 1：你想改某条铁律

你说：**"把铁律 #2 的'应'改成'必须'，然后提交"**

我：
1. 找到铁律 #2
2. 告诉你改之前是什么样、改之后是什么样
3. 等你确认
4. 写入文件
5. `git add` + `git commit -m "fix: 铁律#2 措辞强化"` + `git push`

### 场景 2：你想加一条新约束

你说：**"法规新增了一条 H-62，要求样本保存温度不超过 4 度"**

我：
1. 找到 `指导原则约束汇总.md`
2. 按格式加 H-62
3. 问你"这条约束影响哪些章节？"
4. 自动同步到 `约束分发清单.md`
5. 提交

### 场景 3：分支协作

你说：**"我在 feat/iron-rules-update 分支上改"**

我：
1. 自动 `git checkout -b feat/iron-rules-update`
2. 在这个分支上做所有改动
3. 提交到该分支
4. 你 review 后说"合并到 main"，我自动 merge

## 给非技术同事的三条建议

1. **不要害怕改坏**——所有改动都在 Git 里，错了随时回滚
2. **不要省略确认**——AI 改之前会给你看 diff，多看一眼能少踩坑
3. **不要自己跑命令**——所有 Git/Python 操作让 AI 帮你执行

## 故障排查

| 问题 | 解决 |
|------|------|
| AI 说不存在项目 | 确认 `projects/<项目名>/` 目录已创建 |
| 提取的字段缺失 | 检查 IFU .md 文件中是否包含对应章节 |
| V-05 报告有 ❌ | AI 会列出具体约束和修复建议，按建议改 |
| .docx 格式异常 | 用 Word 打开看效果，必要时用 pandoc 二次处理 |
